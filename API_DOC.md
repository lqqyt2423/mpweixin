接口示例以及说明
-------------

#### 接口示例

```json
{
  "status": 0,
  "metadata": {
    "count": 200,
    "totalPages": 20,
    "currentPage": 1,
    "perPage": 10
  },
  "message": "请求成功",
  "data": { "name": "Felix Liu" },
  "errors": [],
}
```

#### 接口说明

``` bash
# status 说明:
#     >= 2: 大于2的状态为接口调用错误, 且错误需要前台显示
#     1: 接口调用成功, 并且需要显示成功信息
#     0: 接口调用成功, 但是不显示成功信息
#     <= -1: 接口调用错误, 但是不显示错误信息
#     -1000: 无效的时间戳, 此时应根据服务器返回的时间戳(timestamp)校准, 并重试失败的接口
#     -1001: 签名无效, 开发状态下, 可以通过比对 客户端签名和服务端签名 调试

# metadata 说明: 一些额外的元数据, 通常包含分页或请求元数据等信息

# 错误处理, HTTP Status 状态码 及 说明
#     1. 200 ~ 299   请求成功
#     2. 401   认证错误, 需要用户登陆操作
#     3. 403   请求校验失败, Authorization 值错误 或者不存在
#     4. 404   请求资源不存在或者接口不存在
#     5. 422   请求参数有错, 可以通过查看返回数据获知那些参数缺失或者格式错误
#     6. 500   服务器错误, 通知接口管理员处理
```

``` bash
# 通用接口说明:
#    1. 所有时间格式均为 [ISO 8601](http://en.wikipedia.org/wiki/ISO_8601)
#    2. 所有 POST, PUT, DELETE 操作, 如无特殊说明均需要登录才能访问
#    3. 对于列表类接口, metadata 中会包含服务器的时间戳以及分页信息, 如下
#       {
#         "timestamp": "2015-10-22T14:48:54.874Z",
#         "method": "POST",
#         "endpoint": "/v1/users/sign_in",
#         "count": 200,
#         "page": 1,
#         "perPage": 10
#       }
#       其中分页信息  page >= 1, 1 <= perPage <=100
#    4. 所有距离相关的字段, 单位均为 米(m)
#
# 通用返回字段说明:
# createdAt: 创建时间
# updatedAt: 更新时间
```
